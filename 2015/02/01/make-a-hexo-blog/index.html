使用Github + hexo 搭建自己的博客

  • 博客
  • 教程

一直纠结在哪里写博客，话说之前是长期在CSDN进行写作的，后来发现好像有自己的域名有自己的博客是个很高(zhuang)级(bi)的事情，于是自己也想搭一个。最开始用的是Jekyll，后来觉得主题都不是很好看，知道发现hexo，这个Pacman的主题就很符合我的胃口，果断尝试使用！

由于网上的文章写的都比较好了，所以本文不是一个完整的教程，只是记录一下自己在配置过程中遇到的坑，给大家以帮助。

hexo的台湾友人写的一款基于Node.js的静态博客引擎，Github Pages是Github上用来展示个人网页的页面。不足的是，Github Pages只能托管静态网站，无法使用数据库等工具保存我们的界面。

hexo就是一个这样的转换引擎，允许在书写博客的时候使用MarkDown语法，添加好适当的主题后，hexo自动生成html并组织成网站模式，最后还可以自动push到Github上，简直不能再赞。

以下内容请先阅读一篇写的非常好的文章hexo你的博客，本文只是它的补充。

##环境准备

安装Node.js
到Node.js官网上下载一个安装包，直接安装即可。我使用的版本是node-v0.10.36-x64.msi.

安装Git
Git的客户端我喜欢用msysgit，既有比较清晰的结构图，也有方便的命令行。在这里主要使用它的命令行。在这里我使用的是exe安装包，安装好了以后环境就配置好了。

书写MarkDown
MarkDown是一个强大且简洁的富文本语法。如果喜欢即时查看写作的效果，就需要使用一个MarkDown客户端。开始我喜欢使用MarkDownPad，不过因为高级版要收费，而且安装空间不菲，最终放弃。现在我使用的是作业部落的在线编辑器，非常强大，有兴趣的同学可以试一试。

##使用taobao.org提供的npm镜像

NPM（node package manager），通常称为node包管理器。顾名思义，它的主要功能就是管理node包，包括：安装、卸载、更新、查看、搜索、发布等。它的官网是https://www.npmjs.com/ ， 有兴趣的同学可以上去看看有什么包。
国内的网络会严重影响NPM的下载速度，无意中发现npm.taobao.org有一个npm镜像，可以解决网络的问题。
打开刚才安装的Git客户端里的Git Bash，输入：

$ npm install -g cnpm --registry=https://registry.npm.taobao.org

产生一个新命令cnpm，自动到镜像站下载文件，先用命令切换到想要做blog的文件夹，比如我用了E:\blog，然后：

$ cnpm install hexo

稍等片刻即可安装好hexo插件。到计算机–环境变量里将PATH中加入：

E:\blog\node_modules\hexo\bin

hexo就基本安装完成了

##建立hexo项目

首先将目录改为E:\blog，然后再Git Bash里输入：

$ hexo init

也可以在其它目录中使用命令完成:

$ hexo init <folder>

使用目前的配置文件创建博客内容，此时静态页面会保存在hexo/public目录：

$ hexo generate

使用命令行进行本地启动：

$ hexo server

以上命令一条龙走下来就可以生成一个网站，浏览器输入http://localhost:4000查看网站的预览。
如果预览不成功，问题表现为只显示index的源代码，说明还没有安装好，在Git Bash中输入：

$ npm install

然后重新重复上述步骤

##书写文章

具体内容请参考hexo你的博客。
文章保存在source目录下，以.md的格式进行保存，使用作业部落的编辑器写好以后直接复制进去就好，每个文章有tag，这里不能使用,或者空格来分割，要使用yaml格式,如下：

title: 开始学习SSH
date: 2015-01-31 22:08:59
tags: 
    - SSH
    - JAVA
    - 服务器

##下载themes

本博客使用的theme是Pacman，想要下载，先将运行目录改为E:\blog,输入：

$ git clone https://github.com/A-limon/pacman.git themes/pacman

更改blog/_config.yml中的代码如下：

theme: pacman

##添加友情链接
更改pacman目录下的_config.yml文件，在widgets下添加一行links：

widgets: 
- category
- tag
- rss
- links

然后更改/layout/_widget/links.ejs中的网址内容即可。

##添加多说组件

由于搭建的是一个静态网站，所以本身是必定不能实现评论等功能的。外置的js评论插件有Diuqus和多说，由于网络的原因，国内一般使用多说，Pacman主题是默认支持多说的，这里的处理比较方便：

首先使用QQ或者微博登录多说，注册后获取自己的个人域名，（不知道多说为什么要把这个界面弄得这么隐蔽）比如我的页面是icyfox.duoshuo.com

打开pacman/_config.yml，更改duoshuo代码如下：

#### Comment
duoshuo: 
    enable: true  ## duoshuo.com
    short_name: icyfox   ## duoshuo short name.

重新部署后就可以看到每个文章中都有了评论的界面，在icyfox.duoshuo.com中可以管理我的评论。

##布署到Github

hexo强大的到可以自动实现Github的布署。这里先在Github上创建一个新的Git，在这里我命名为：https://github.com/icyfox-bupt/icyfox-bupt.github.io

在blog/_config.yml中，更改Github配置为：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
    type: github
    repository: https://github.com/icyfox-bupt/icyfox-bupt.github.io.git
    branch: master

改完后执行命令：

$ hexo deploy
#或者
$ hexo d

中间会让输入github的用户名和密码，通过后就可以到icyfox-bupt.github.io上查看自己的文章了。

##结束

这篇博客欠缺的东西还有很多，但是已经用了我3个小时的时间来组织，所以大牛们写出一个完整的教程来更是不容易，在此还是感谢上面教程的作者，本文只是一个补充，如果想要看详细的内容，请查看上面博客的文章，也欢迎大家和我一起交流。
